# Guide Template

Use this structure for each per-flow how-to guide.
Write as you capture each step — don't batch to the end.

---

## Per-Flow Guide Structure

```markdown
# How to [verb] [noun]

> Section: [navigation path, e.g. "Dashboard > Settings > Integrations"]
> Steps: [N]
> Last updated: [YYYY-MM-DD]

## Overview

[1-2 sentences: what this accomplishes and when you'd use it.]

## Before You Start

- [Prerequisite: must be logged in, must be on a specific page, need a specific role, etc.]
- [Any setup: "You need at least one project created first"]

## Steps

### Step 1: [Action title]

![Step 1](../images/{NN}-{flow-slug}-step-01.png)

[Action instruction — see Writing Style Rules below.]

### Step 2: [Action title]

![Step 2](../images/{NN}-{flow-slug}-step-02.png)

[Action instruction.]

[Continue for all steps...]

## Result

![Result](../images/{NN}-{flow-slug}-result.png)

[What you should see when done. Confirm the action succeeded.]
```

## Writing Style Rules

Follow these rules in every guide to maintain consistency.

### Voice and tense
- **Second person** ("You"), **present tense** ("The dialog opens")
- Address the reader directly: "You'll see a confirmation message"

### Step instructions
- **Start every step with a verb:** "Click", "Enter", "Select", "Navigate to", "Scroll to", "Toggle", "Drag"
- **One action per step** — atomic. If it involves two clicks, it's two steps.
- **Bold exact UI text:** Click **Create Project**, Select **Team Settings**
- **Be specific about location:** "in the top-right corner", "in the left sidebar", "below the table", "in the modal that appears"

### Annotation colours
- **Primary (orange):** `#FF6B35` — used for the main element being highlighted in a step
- **Secondary (blue):** `#2196F3` — used for secondary elements referenced in the same step
- On dark-themed sites, consider lighter alternatives: primary `#FF9966`, secondary `#64B5F6`

### Image naming convention
- Format: `{NN}-{flow-slug}-step-{SS}.png`
  - `NN` = two-digit flow number (matches guide file prefix)
  - `flow-slug` = kebab-case flow name (e.g., `create-project`, `invite-member`)
  - `SS` = two-digit step number within the flow
- Result screenshots: `{NN}-{flow-slug}-result.png`
- Examples: `01-create-project-step-01.png`, `01-create-project-result.png`

### Things to avoid
- Don't use "simply" or "just" — it implies the step is trivial
- Don't combine multiple actions: "Click X and then enter Y" should be two steps
- Don't describe what the user already sees — describe what to DO
- Don't use passive voice: "The button should be clicked" → "Click the button"
